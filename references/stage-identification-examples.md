# Stage Identification Examples

这个文件提供第一版 `impl-phase-coach` 的判阶段案例。目标不是覆盖所有情况，而是帮助另一个 Codex 在真实任务里更稳定地选对“当前最阻塞推进的最早阶段”。

## Case 1: 刚接到任务，不确定先做什么

用户请求：
“我要开始做这个功能，但你先别直接写，先告诉我我现在该处在哪个实现阶段。”

建议判断：`阶段 0 / 阶段识别与边界冻结`

原因：

- 用户最主要的问题还不是模块、状态或代码实现，而是先确认当前阶段边界。
- 这时如果直接进入代码，会跳过阶段冻结。

为什么不是别的阶段：

- 不是 `阶段 1`，因为文件边界和接口还没有成为最直接的问题。
- 不是 `阶段 3`，因为还没有到“补关键实现”的时机。

## Case 2: 主要纠结文件边界和函数签名

用户请求：
“这个 handler 到底放在哪个文件，返回结构要不要现在就定下来？”

建议判断：`阶段 1 / 模块与接口冻结`

原因：

- 用户当前问题集中在职责归属、文件边界和输入输出契约。
- 如果这些不先冻结，后续状态设计和实现都会漂。

为什么不是别的阶段：

- 不是 `阶段 0`，因为用户已经明确进入实现讨论。
- 不是 `阶段 2`，因为状态语义还不是最主要矛盾。

## Case 3: `DENIED` 和 `FAILED` 混在一起

用户请求：
“我现在最乱的是状态返回，拒绝、失败和空结果都混在一起了。”

建议判断：`阶段 2 / 状态与最小闭环冻结`

原因：

- 当前核心矛盾是状态语义和值域不稳定。
- 这类问题不先收敛，测试矩阵和上层分支都不可靠。

为什么不是别的阶段：

- 不是 `阶段 1`，因为虽然接口有关，但真正阻塞推进的是状态语义。
- 不是 `阶段 3`，因为此时还不应该先补实现细节。

## Case 4: 用户已经写了一半代码，希望你补关键位置

用户请求：
“我把骨架写了，你只补最关键那部分，别一次性全写完。”

建议判断：`阶段 3 / 当前阶段实现与补码`

原因：

- 用户已经接受当前阶段边界，接下来要做的是局部实现和定点补码。
- 此时应该优先给关键位置示例代码，并提供 A/B 两种推进方式。

为什么不是别的阶段：

- 不是 `阶段 1` 或 `阶段 2`，除非你从用户代码里发现边界或状态已经明显漂移到无法继续。
- 如果发现漂移，应该先降回更早阶段，而不是硬补实现。

## Case 5: 功能大致通了，但排错很难

用户请求：
“功能现在能跑，但日志太弱了，出了问题我根本定位不到。”

建议判断：`阶段 4 / 运行观测补齐`

原因：

- 主要问题已经不是主逻辑能不能工作，而是缺少可追踪的运行观测信息。
- 这时要优先检查 `request_id`、事件链、错误码和审计字段。

为什么不是别的阶段：

- 不是 `阶段 3`，因为重点不是补业务逻辑。
- 不是 `阶段 5`，因为在可观测性不足时，验收判断也不可靠。

## Case 6: 用户要你判断这一阶段能不能结束

用户请求：
“你别再往下写了，先看这一阶段测试矩阵够不够，能不能冻结。”

建议判断：`阶段 5 / 验收与阶段复盘`

原因：

- 用户当前关注点是退出条件、测试矩阵和版本成立标准。
- 这时应该围绕验收标准做判断，而不是继续扩写实现。

为什么不是别的阶段：

- 不是 `阶段 3`，因为现在不是继续补实现的时候。
- 不是 `阶段 4`，除非验收失败的根因是观测完全缺失。

## Conflict Rule

如果一个请求同时像两个阶段，默认使用下面的规则：

1. 先找最早且最阻塞推进的阶段。
2. 如果更早阶段没冻结，不要跳去后面的阶段。
3. 如果用户明确要求只处理当前阶段最小闭环，就只回答那个闭环。

## Subphase Rule

如果用户已经处在某个主阶段中，随后给出：

- 当前阶段的测试反馈
- 当前阶段的调试结果
- 当前阶段的冻结审查意见
- 当前阶段内的局部疑惑或纠偏请求

默认把这些视为“当前主阶段内的衍生子流程”，不要直接判成一个新的主阶段。

只有在反馈明确说明：

1. 当前主阶段判断本身错了，或
2. 当前最阻塞推进的问题已经换成别的核心冻结项

才允许重判主阶段。

例子：

- 用户原本在 `阶段 5 / 验收与阶段复盘`
- 用户随后说“这轮测试里我发现 `DENIED` 和 `FAILED` 的边界还有点乱，你帮我看”

默认处理方式：

- 先把它视为 `阶段 5` 内的验收纠偏，不要立刻把整件事改判为 `阶段 2`
- 只有当你确认当前主要阻塞已经从验收层退回到状态层冻结时，才把主阶段下调

## Downgrade Rule

如果用户表面上请求的是更后面的阶段，但你从材料里发现更早阶段并没有成立：

1. 明确指出你为什么要降回更早阶段。
2. 只说明和当前阻塞有关的那部分，不回到高层战略。
3. 等更早阶段冻结后，再继续后面的阶段。
