# Review Gate Integration

## 目标

这份文档说明 `impl-phase-coach` 主 skill 如何软接入 `review_gate`。

接入口径：
- `review_gate` 是当前主阶段内的验证子流程
- 它不是新的主阶段
- 它的结果用于辅助判断是否建议进入下一阶段

## 何时建议复盘

默认只在下面情况建议：
- 用户像是看懂了，但还不能稳定输出当前阶段内核和输出手法
- 准备从当前阶段进入下一阶段前，需要验证理解是否稳定
- 用户主动要求复盘、拷打式追问、阶段内核检查或学习建议

默认不要强制进入复盘。

## 主流程中的输出方式

当满足建议条件时，在主 skill 回复末尾追加一个 `复盘建议（可选）` 块：

1. 为什么现在建议复盘
2. 当前建议模式
3. 用户可选入口：
   - `跳过复盘`
   - `简单复盘`
   - `深入复盘`

## 复盘结果如何回流主流程

### `pass`

表示当前阶段理解已稳定：
- 可以继续建议进入下一阶段
- 但仍要结合当前阶段退出条件，不要只因为复盘通过就自动跳阶段

### `continue_probing`

表示当前阶段理解仍不稳定：
- 留在当前阶段
- 继续做追问或纠偏
- 不建议进入下一阶段

### `redirect_to_learning`

表示存在系统性知识缺口：
- 留在当前阶段
- 优先给学习建议
- 不建议进入下一阶段

## 项目侧调用建议

项目侧如果要真正执行复盘，优先调用：
`run_review_workflow()`

推荐最小落地产物：
- HTML：给人类直接看
- snapshot JSON：给项目留痕和后续维护

## 不要做的事

- 不要把复盘升格成新的主阶段
- 不要让复盘替代主阶段目标、产物、退出条件说明
- 不要把复盘做成每轮强制阻塞
- 不要让 HTML 或 snapshot 替代结构化阶段判断
